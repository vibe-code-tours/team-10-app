# spec-driven-development အသုံးပြုနည်း

ပရောဂျက်အသစ်စလုပ်တာ၊ Feature အသစ်ထည့်တာ ဒါမှမဟုတ် အပြောင်းအလဲကြီးကြီးမားမား လုပ်တော့မယ်ဆိုရင် ကုဒ်မရေးခင် **အစီအစဉ်/လိုအပ်ချက် (Specification - Spec) ကို အရင်ရေးဆွဲပေးမယ့်** Skill ဖြစ်ပါတယ်။

## အလုပ်လုပ်ပုံအဆင့်ဆင့် (The Process)

၁။ **Specify (သတ်မှတ်ချက်များ ရှင်းလင်းခြင်း)**: အကြမ်းဖျင်း ရည်ရွယ်ချက်ကို အရင်ဆုံး မေးမြန်းပါမယ်။ လိုအပ်တာတွေ ထပ်မေးပြီး ရှင်းလင်းအောင် လုပ်ပါမယ်။
၂။ **Write the Spec (Spec ရေးသားခြင်း)**: လိုအပ်ချက် မှတ်တမ်း (Spec) ကို ရေးသားရာမှာ အောက်ပါအချက်တွေ ပါဝင်ရပါမယ်:
   - **Objective**: ရည်ရွယ်ချက်။
   - **Tech Stack**: အသုံးပြုမည့် နည်းပညာများ။
   - **Commands**: အသုံးပြုရမည့် Command များ။
   - **Project Structure**: ဖိုင်နှင့် Folder တည်ဆောက်ပုံ။
   - **Code Style**: ကုဒ်ရေးသားမှု ပုံစံ။
   - **Testing Strategy**: စမ်းသပ်မည့် နည်းလမ်းများ။
   - **Boundaries**: ဘာတွေလုပ်ရမယ်၊ ဘာတွေမေးရမယ်၊ **ဘာတွေ လုံးဝ (လုံးဝ) မလုပ်ရဘူး** ဆိုတဲ့ ကန့်သတ်ချက်များ။
၃။ **Follow the Rule (စည်းမျဉ်းလိုက်နာခြင်း)**: ဒီ Skill ဟာ ပရောဂျက်ရဲ့ "Proposal First" (အစီအစဉ် အရင်တင်ပြပါ) ဆိုတဲ့ စည်းမျဉ်းကို တင်းကျပ်စွာ လိုက်နာပါတယ်။ ရေးဆွဲထားတဲ့ Spec ကို **User က အတည်ပြုပြီးမှသာလျှင်** ကုဒ်ကို စတင်ရေးသားမှာ ဖြစ်ပါတယ်။
