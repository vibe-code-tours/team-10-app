# /spec-driven-development

Write a structured specification before writing any code. Code without a spec is guessing.

## The Process
1. **Specify**: Start with a high-level vision. Ask clarifying questions. Surface assumptions immediately.
2. **Write the Spec**: The spec must cover Objective, Tech Stack, Commands, Project Structure, Code Style, Testing Strategy, and Boundaries (what to do, what to ask, what never to do).
3. **Follow the Rule**: This strictly aligns with the "Proposal First" rule (Rule 2) of this project. Do not code until the spec is approved.
