# save-memory အသုံးပြုနည်း

အလုပ် (Session) တစ်ခု မပြီးဆုံးခင်မှာ ဆွေးနွေးခဲ့တဲ့ ဆုံးဖြတ်ချက်တွေ၊ ပြောင်းလဲမှုတွေ၊ သတိထားရမယ့် အချက်တွေကို နောက်ထပ် Session အတွက် ကျန်ရှိနေအောင် `.agents/memory/` ဖိုင်တွေထဲမှာ မှတ်တမ်းတင် သိမ်းဆည်းပေးမယ့် Skill ဖြစ်ပါတယ်။ (`/start` ခေါ်တဲ့အခါ ဒီမှတ်တမ်းတွေကို ပြန်ဖတ်မှာ ဖြစ်ပါတယ်)

## အလုပ်လုပ်ပုံအဆင့်ဆင့် (Steps)

၁။ **MEMORY.md တွင် မှတ်တမ်းတင်ခြင်း (MEMORY.md entry)**
- `.agents/memory/MEMORY.md` ဖိုင်ရဲ့ အပေါ်ဆုံးမှာ အောက်ပါ အချက်အလက်များဖြင့် မှတ်တမ်းတင်ပါမယ်:
  - **Changed:** ဘာတွေ ပြောင်းလဲ/လုပ်ဆောင်ခဲ့သလဲ။
  - **Why:** ဘာကြောင့် ပြောင်းလဲရသလဲ (ဖြေရှင်းလိုက်တဲ့ ပြဿနာ)။
  - **Gotchas:** နောက်လူတွေ သတိထားရမယ့် ထူးခြားချက်တွေ၊ မှားတတ်တဲ့ အရာတွေ။
  - **Open threads:** မပြီးသေးတဲ့၊ ဆက်လုပ်ဖို့ကျန်နေတဲ့ အရာတွေ။

၂။ **ဆုံးဖြတ်ချက်မှတ်တမ်း (Decision record - ADR)**
- အရေးကြီးတဲ့ နည်းပညာရေးရာ ဆုံးဖြတ်ချက်တွေ၊ စည်းမျဉ်းအသစ်တွေ ပါလာခဲ့ရင် `.agents/memory/decisions/` အောက်မှာ ADR (Architecture Decision Record) အသစ်တစ်ခု ဖန်တီးပြီး မှတ်တမ်းတင်ပါမယ်။

၃။ **ဖွဲ့စည်းပုံဆိုင်ရာ မှတ်တမ်း (Structure docs)**
- ဖိုင်အသစ်တွေ ဖန်တီးတာ၊ ရွှေ့တာ၊ ခွဲထုတ်တာတွေ လုပ်ခဲ့ရင် `.agents/architecture/project_mapping.md` နှင့် လိုအပ်ပါက `docs/ARCHITECTURE.md` ကို အပ်ဒိတ်လုပ်ပါမယ်။

၄။ **စည်းမျဉ်းအသစ်များ (New convention)**
- ကုဒ်ရေးသားခြင်းဆိုင်ရာ စည်းမျဉ်းအသစ်တွေ ပေါ်လာရင် သက်ဆိုင်ရာ `.agents/` အောက်က ဖိုင်တွေမှာ ဖြည့်စွက်ပါမယ်။

၅။ **လင့်ခ်များ စစ်ဆေးခြင်း (Link check)**
- မှတ်တမ်းဖိုင်တွေထဲက လင့်ခ်တွေ (ဥပမာ ဖိုင်နေရာရွှေ့လိုက်လို့ လင့်ခ်အသေဖြစ်သွားတာမျိုး) ကို ဒီ session မှာ ပြင်ထား/ထည့်ထားတဲ့ link တွေ ကိုယ်တိုင်ပြန်ဖွင့်ကြည့်ပြီး စစ်ဆေးပါမယ် (project မှာ automated link checker မရှိသေးပါ)။

၆။ **အပြောင်းအလဲများကို ပြသခြင်း (Show the diff)**
- User ကို Commit မလုပ်ခင် ဘာတွေပြောင်းလဲသွားလဲဆိုတာကို `git diff` ဖြင့် ရှင်းလင်းစွာ ပြသပါမယ်။ (မှတ်တမ်းများကို အင်္ဂလိပ်ဘာသာဖြင့် ဦးစွာရေးပြီး မြန်မာဘာသာဖြင့် တွဲလျက်ရေးသားရပါမည်။)
