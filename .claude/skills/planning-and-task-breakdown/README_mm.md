# planning-and-task-breakdown အသုံးပြုနည်း

ကြီးမားတဲ့ ပရောဂျက် သို့မဟုတ် Feature အသစ်တစ်ခု လုပ်ဆောင်ရတော့မယ့်အခါ အဲ့ဒီအလုပ်ကို အသေးစိတ် Task လေးတွေအဖြစ် စနစ်တကျ ခွဲခြမ်းစိတ်ဖြာပေးမယ့် Skill ဖြစ်ပါတယ်။

## အလုပ်လုပ်ပုံအဆင့်ဆင့် (The Process)

၁။ **Enter Plan Mode (အစီအစဉ်ဆွဲခြင်း)**: ကုဒ်စတင်မရေးသေးဘဲ သတ်မှတ်ချက်များ (Spec) နှင့် လက်ရှိ ကုဒ်များကို ဖတ်ရှုလေ့လာပါမယ်။
၂။ **Dependency Graph (ချိတ်ဆက်မှုများ ဆွဲခြင်း)**: ဘယ်အရာက ဘယ်အရာအပေါ် မှီခိုနေလဲဆိုတာကို ကြည့်ပြီး အခြေခံအကျဆုံး အပိုင်းတွေကို အရင်ဆုံး လုပ်ဖို့ အစီအစဉ်ဆွဲပါမယ်။
၃။ **Slice Vertically (ဒေါင်လိုက်ခွဲထုတ်ခြင်း)**: အလုပ်တစ်ခုလုံး ပြီးမြောက်ဖို့အတွက် လိုအပ်တဲ့ (UI + API + DB) အဆင့်သုံးဆင့်လုံး ပါဝင်တဲ့ အလုပ် (Feature path) တစ်ခုကို တစ်ဆက်တည်းဖြစ်အောင် ခွဲထုတ်ပေးပါမယ်။
၄။ **Write Tasks (Task များ ရေးသားခြင်း)**: Task တစ်ခုစီတိုင်းဟာ တာဝန်တစ်ခုတည်းကိုပဲ ထမ်းဆောင်ရမယ် (Single-responsibility) ဆိုတဲ့ စည်းမျဉ်းကို လိုက်နာဖို့ သေချာစေပါမယ်။ ဖိုင်တစ်ခုမှာ ကုဒ်လိုင်းရေ ၃၀၀/၅၀၀ ထက် မကျော်လွန်စေဖို့ လိုအပ်ပါက ဖိုင်ခွဲထုတ်မယ့် အစီအစဉ်တွေကိုပါ ကြိုတင်ထည့်သွင်း စဉ်းစားပါမယ်။
